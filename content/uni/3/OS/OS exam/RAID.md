# RAID
### Концепции
Ввод вывод к нескольким дискам может происходить параллельно
С увеличением количества диском надёжность падает 
RAID — несколько дисков, которые ОС логически объединяет в один
- Данные распределены по всем дискам
- Может использоваться дополнительная емкость для хранения контрольной информации
- Один блок данных может находиться на нескольких дисках 
- Один запрос может также быть выполнен параллельно
- Необходимо учитывать производительность адаптеров и шин подключения к адаптерам, а также шины IO (PCI, PCIe)
### Concatenation & Striping
>RAID 0 Объединяет диски в один том большего размера

Concatenation — подключили несколько дисков, объединяем их в один с помощью ПО (последовательно). Но производительность одного не вырастет 

Stripe — Делим диски на несколько блоков одинакового размера ( Stripe unit ). Позволяет читать файл, и часть данных попадет на один диск, часть — на другой и часть — на третий. 
### Зеркалирование, RAID 1, 1+0, 0+1
> RAID 1 Объединяет диски, копируя блоки

Добавляем надёжность путём копирования данных с блоков дисков (по сути бэкап дисков).
RAID 0+1 - RAID 1 из RAID 0 (страйпим данные по дискам, потом
зеркалируем)
Отказ диска -> т.к. группы большие, страйп валится целиком; надо
его вырубать, заменять диск, собирать новый страйп и
синхронизировать с зеркалом
RAID 10 - RAID 0 из RAID 1 (зеркалируем, потом страйпим)
Отказ диска -> т.к. группы маленькие, валится только диск;
заменяем его и синхронизируем

RAID 2 и RAID 3 
в реальной жизни почти не встречаются
- Диски синхронизированы между собой по позициям головок
- Может использоваться очень маленький stripe unit
- На практике используются редко 
- Запись на все диски одновременно
Используются всякие подсчёты четности, коды Хемминга и тд.

### RAID 4-5-6
RAID 5 идея: на разных дисках подсчитывать чётность для stripe unit-ов. 
- высокая надежность. избыочность disk+1
- дополнительные вычисления при записи, и при чтении в случая сбоя диска 
- пропускная способность на чтение как у raid 0 и меньше, чем одинарная на запись
- скорость обработки запросов для чтения как у RAID 0 и меньше, чем одинарная на запись
RAID 4: выделенный диск с четностью
RAID 6: двойная размазанная четность, по разным алгоритмам, избыточность disk+2
