##### Protocol
- 1/3 краткое изложение основных идей материала;
- 1/3 ваша личная оценка основных идей материала (их полезность, актуальность, достоверность);
- 1/3 ваши рассуждения/мнение на поднятую в материале проблему/тему.

Текст рецензии должен включать ответы на следующие вопросы:

- Удалось ли автору эссе изложить основные идеи материала?
- Удалось ли автору сформулировать аргументированную оценку идей материала?
- Удалось ли автору сформулировать аргументированное мнение касательно поднятой в материале проблемы?
- Общее впечатление от прочитанного эссе.
Рецензия включает формирование рейтинга эссе, полученных студентом на рецензирование по обозначенным критериям

##### Listening
“Ideas that require people unlearn what they’ve learned and think in new ways there’s often enormous amount of resistance”
1. direct manipulation of data – there wont be any markup languages to describe data manipulation and data structures.
2. programming using goals – SNOBOL regex example, logic programming. “if the program knows the apple is red, then when we say “i want something red”, it should be able to answer – apple”
3. “i am pretty sure we’re not gonna get API’s in the future” – we’re gonna have programs dynamically figuring out how to talk to each other. And that’s gonna require programming using goals 
4. Spacial representations. Flowcharts (GRAIL), linear data (NLS), project structure with classes, methods implementation, etc. → spacial representation. “I am totally sure that in 40 years we won’t be writing code in text files”. Dynamic interaction with the PC. You will never experience any kind of delay or lag.
5. Parallel programming. Today programs are sequences of instructions. Microprocessor – entire processor on a single piece of silicon.
